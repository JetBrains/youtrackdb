# Golden apply-tier table for all registered PageOperation classes (YTDB-1203, Track 03).
#
# This file is the durable home of the tier census. Any change to it must be a deliberate,
# reviewed reclassification against the SC-P / SC-R / G2 side conditions documented on
# ApplyTier. ApplyTierGoldenTableTest fails on any drift between this table and the
# classes registered by PageOperationRegistry.
#
# Columns (pipe-separated):
#   1 class            fully qualified concrete PageOperation class name
#   2 tier             declared ApplyTier (must equal <class>.applyTier())
#   3 establishes      yes if the op establishes in-place content targeted by a same-commit
#                      pointer (SC-P obligation; fresh-page establishment is immune via
#                      NEW-force)
#   4 primaryPublish   yes if the op is the publish making a created/relocated entry's copy
#                      reachable (SC-R obligation when retiredBy is set)
#   5 publishedBy      for establishing rows: the same-commit pointer op targeting the
#                      established content ('-' otherwise)
#   6 retiredBy        for primary-publish rows: the same-commit op retiring the relocated
#                      entry's old copy on a DIFFERENT page ('-' when the retire is
#                      same-page-atomic, in a different commit, or not applicable - explain
#                      in the note)
#   7 colocations      op classes that can land on the same page in one commit (max-merge
#                      lift set; comma-separated, '-' if none)
#   8 note             census gist; UNORDERED rows must start with 'DEAD:' citing the
#                      deadness evidence
