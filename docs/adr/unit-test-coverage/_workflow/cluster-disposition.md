Generated by Phase B Step 0 (pin-to-disposition table + 22a anchor pre-flight).

This artifact maps every `*DeadCodeTest.java` shape pin in the `core/src/test/java`
tree to one of three dispositions:

- **delete-in-22b cluster: <name>** — production target is dead per PSI find-usages
  across all-scope (core, server, embedded, driver, tests modules; including
  `*IT.java`); the deletion satisfies the hybrid policy (zero production callers,
  not api/-reachable, not an SPI service / abstract base / catchable exception
  with external consumers, no cross-module coordination).
- **defer-to-22c** — at least one PSI hybrid-policy criterion fails (live consumer,
  abstract base with subclasses, SPI / `META-INF/services`-registered, exception
  type catchable by external code, api/-reachable, etc.). Production source
  untouched in 22b; the YTDB issue + WHEN-FIXED rewrite happens in 22c.
- **pin-maintenance** — PSI shows the production target is ALIVE despite the
  pin's `DeadCode` suffix; the pin is renamed (drop `DeadCode`) or its assertions
  retuned in 22b without touching production code.

The dispositions also serve as 22c's filter input — 22c reads this table to
determine which WHEN-FIXED markers still need YTDB issues.

## Methodology / evidence

PSI find-usages for every pin's production target was run via
`ReferencesSearch.search(psiClass, GlobalSearchScope.allScope(project))`
inside the IntelliJ Platform's PSI engine. Reference counts are bucketed by
filename suffix into:

- **prod refs** — non-test source files (production code in `core/`, `server/`,
  `embedded/`, `driver/`).
- **non-dead-test refs** — test source files NOT ending in `DeadCodeTest.java`,
  excluding `*IT.java`.
- **dead-test refs** — test source files ending in `DeadCodeTest.java` (the
  pins themselves).
- **IT refs** — test source files ending in `IT.java` (integration tests).

The full per-pin JSON output is preserved at
`.coverage/pin-disposition-758.json` and
`.coverage/multi-target-pin-disposition-classlit-758.json` (gitignored
build artefacts; the table above carries the durable evidence chain).

PSI safe-delete formal classification (`mcp-steroid://ide/safe-delete`) is
re-run by each cluster commit (Steps 1-11) before drafting the deletion;
this Step 0 artifact is the index that makes those per-cluster runs cheap.
