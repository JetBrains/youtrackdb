# Research Log — reviewers-loose

## Initial request

Fix **YTDB-1147** — *Single-step high workflow-only track loses its four deferred workflow reviewers* (Bug, Major, dev-workflow).

**Source:** branch `state-ledger-fix`, commit `e3158fe135845ac7e2d5c23719bca93fcef997a9`. Surfaced in Phase B of a minimal-tier, single-step, `risk:high`, workflow-only track.

**Symptom (verbatim from the issue).** During Phase B of a minimal-tier, single-step, `risk:high`, workflow-only track, step-level dimensional review per `review-agent-selection.md` §Step-level vs track-level routing fires only the `{hook-safety, prompt-design}` subset gated by file globs — here just `review-workflow-hook-safety` matched (a `.py` test), because `track-review.md` is a workflow rule, not a prompt. The other four workflow reviewers (consistency, context-budget, writing-style, instruction-completeness) defer to the track pass. But `code-review-protocol.md` §Single-step tracks and `track-code-review.md` §Single-Step Track skip the Phase C code-review fan-out when a single-step track's sole step is `risk:high`. The two rules together leave the substantive `track-review.md` edit reviewed by zero of the four deferred reviewers. Running the full selection anyway at the step caught a real should-fix that the literal rule would have dropped.

**Why it's a problem.** The single-step-high Phase C skip's stated premise — "the step-level review already covered the identical diff … fully reviewed in Phase B" — is false, because step-level review runs only a subset and the rest were deferred to a track pass that never runs.

**Proposed fix (from the issue).** In `code-review-protocol.md` §Single-step tracks (cross-referenced in `review-agent-selection.md` §Step-level vs track-level routing), add a rule: when a single-step track's sole step is `risk:high`, the step-level review must run the full track-pass-equivalent reviewer selection (not just the `{hook-safety, prompt-design}` step-level subset), because the Phase C track pass the four deferred reviewers rely on is skipped. The single-step-high Phase C skip is valid only once the full selection has run at the step.

**Acceptance criteria (from the issue).**
- `code-review-protocol.md` §Single-step tracks (or `review-agent-selection.md` §Step-level routing) states that single-step `risk:high` tracks run the full track-pass-equivalent selection at the step.
- A single-step high workflow-only track no longer drops the four deferred reviewers.
- The single-step-high Phase C skip's "fully reviewed in Phase B" premise holds because the full selection ran at the step.
